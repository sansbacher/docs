Some Additional Topics

If you will be editing or creating Scripts you should find a better editor than Notepad, something with Syntax Highlighting will make it much easier to code and spot errors. Some suggestions:

  • Windows includes the PowerShell Integrated Scripting Environment (ISE) - usually installed and ready to go. Try running: ISE.EXE from PowerShell
    Or look in the Start Menu for "Windows PowerShell ISE"
  • Visual Studio Code has solid PowerShell support with the PowerShell extension
  • Notepad++ is also very popular and works with .PS1, .BAT and .CMD files too.
  • Or the smaller, portable and very similar Notepad2, also works with .PS1/.BAT/.CMD files.
  • Unless you already know them, don't try to use Vim or Emacs. If you already know them, feel free to use them

If you run across a VBScript (.VBS file) you can run it with either WSCRIPT.EXE or CSCRIPT.EXE. Wscript is the Windows Scripting Host, and won't open a console, but any output is done via pop-up message boxes. Cscript is the Console Scripting Host - so while a Console opens the output is all at console. Example:
cscript.exe //nologo c:\path\and\NameOfScript.VBS

When pasting single or multiple commands from the web to a Console/Shell (or into a script file) keep in mind that may blogs will mangle the commands. PowerShell.exe and CMD.exe aren't very forgiving at times. So extra spaces need to be removed, or added. Funny characters may need to be trimmed, and those curly "smart quotes" may need to changed to regular "double" or 'single' quotes. Sometimes dashes need to be changed from longer Em- and En-dashes to regular hyphens/dashes. If something still doesn't work: check to see if spaces were added (like Get -Content -Path someFile.txt) or removed (like Get-Content-Path someFile.txt).

Tip: You can type cmd or powershell in the Address Bar of any Windows File Explorer window to open the CMD Command Prompt or a PowerShell Console in that folder, which can save a lot of CD'ing or copy/paste.
